# 株取引シミュレーションアプリ構想まとめ

## 1. 目的

Claude を用いて、日本株の売買判断を自動で行うシミュレーションアプリを作成する。

- 初期資金 X 円を与える
- 一定時刻 t ごとに Claude が売買判断
- 最終時刻 T における現金残高を最大化する
- T 時点で全ポジションを解消済みとする
- 売買履歴を CSV で記録する

対象は以下の取引。

- 現物買い
- 現物売り
- 空売り（売建）
- 買い戻し（返済買い）

---

## 2. 運用モード

### 中期モード（推奨）

- T = 1か月
- t = 毎営業日 9:00 前に 1 回
- 取引回数: 約20回/月

### 短期モード

- T = 1週間
- t = 市場時間中 10 分ごと
- 取引回数: 約150回/週

### 推奨順位

1. 1か月 / 日次
2. 3か月 / 週次
3. 1週間 / 日次
4. 1週間 / 10分足

最初に利益を狙うなら中期モードが最も現実的。

---

## 3. 売買ルール

各判断時点で Claude に以下を渡す。

- 現金残高
- 保有株
- 空売り建玉
- 直近の株価データ
- 財務指標
- ニュース要約
- 過去の取引ログ

Claude は以下のいずれかを出力する。

- BUY
- SELL
- SHORT
- COVER
- HOLD

出力には以下を含める。

- 銘柄コード
- 株数
- 判断理由
- 想定リスク

---

## 4. 約定ルール

### 推奨: 寄付成行

- 9:00 の寄り付きでのみ約定
- 約定価格 = 当日の始値
- 約定しなければその日の注文は失効
- 9:10 以降の約定は認めない

### 未約定時の処理

- 取引なし
- ポジション変更なし
- 翌営業日に再判断

---

## 5. 最終条件

最終時刻 T において

- 現物株はすべて売却
- 空売り建玉はすべて買い戻し
- 最終現金残高のみで評価

目的関数:

max(final_cash)

---

## 6. 取引ログ

CSV 形式で記録。

```csv
date,time,symbol,action,shares,price,status,cash_after,reason
2026-05-11,09:00,7203,BUY,100,2800,FILLED,720000,業績好調
```

---

## 10. 期待リターンの現実的な見積もり

- 年率期待リターン: 0〜10%
- 月次期待リターン: 0〜0.5%
- 月次標準偏差: 4〜6%

100万円を1か月運用した場合:

- 期待値: 100万〜100.5万円
- 1σ範囲: 約95万〜105万円

## 14. コスト

- GitHub Pages: 無料
- GitHub Actions: 多くの用途で無料枠内
- Claude API: 従量課金
- 株価データ: 無料ソースで開始可能

---

## 15. 最終的な推奨構成

- 運用頻度: 毎営業日1回
- 期間: 1か月
- 注文方式: 寄付成行
- 実行環境: GitHub Actions
- UI: GitHub Pages
- AI: Claude API
- 保存: CSV
- 対象: 日本株大型株

---

## 16. 一言でまとめると

GitHub Pages をフロントエンド、GitHub Actions を定期実行基盤、Claude API を意思決定エンジンとして、日本株の日次売買をシミュレーションし、CSV に記録しながら最終資産額を最大化するアプリ。
